Shard - "The OSR of programming languages"

Premise

Flexible, assembly-adjacent language geared towards compile target flexibility, ease of use, and direct control.

No implicit assumptions, everything must be explicitly declared.

Features

- Fine grained low level access to hardware features, like registers, syscalls, and interrupts.

- No "Safety Features" preventing you from doing what you want. Nor stupid stuff. (not mutually exclusive)

- Terse, yet probably clear and concise syntax.

- Architecture details are defined within the standard library, meaning adding support for additional ones doesn't require modifying any compiler source code, and can by done by the user. As long as there's a compatible assembler, of course.

- Shark mascot! Which someone (not pointing fingers, however their name might begin with "j" and end with "esse") hasn't drawn yet :/

Non-Features

- Shard is neither meant to be cross-platform, nor an IR. All architectures are inherently dissimilar, reconsiling them into one cannot be done without compromises

- Planned optimization level is low if not none. If you want to write crazy fast code go use one of these instead: C, Rust, Zig

- Standard library is only for defining the architecture, and writing base wrappers and helpers. You wont find anything over the bare minimum Use libc if you want an expansive standard library.

Novelties

- Type Casting Operators: What began as Shard's answer to float and signed math, has now become one of the core concepts of the type system. This now allows for behaviours similar to methods and generic types in other languages, with practically none of the heavy lifting!

- Self Contained Projects: No need for 20 separate files with layers of build scripts and manifest files. Shard files can contain build scripts, project definitions, packaging/version info on their own through the Tag system!
